PART 6 – Backend Architecture & MVC

✅ 1️⃣ Backend Architecture na enna?

Backend la namma code ellam random ah write panna koodathu ❌

Structure ah organize pannanum ✔

Normal backend structure:

project/ │ ├── config/ ├── controllers/ ├── models/ ├── routes/ ├── server.js └── .env

👉 Idhu than clean architecture.

English Sentence:
A backend architecture organizes code into separate layers for better structure and maintainability.

✅ 2️⃣ Route na enna?

Route na:

Client request receive pannura place

URL handle pannum

Example:

app.get("/users", getUsers)

Inga:

"/users" → Route path

getUsers → Controller function

Client → Route → Controller

English Sentence:
A route handles incoming HTTP requests and directs them to the correct function.

✅ 3️⃣ Controller na enna?

Controller:

Business logic handle pannum

Database kitta pesum

Response send pannum

Example:

export const getUsers = async (req, res) => { const users = await User.find() res.json(users) }

Controller job:

Data process

DB call

Response send

English Sentence:
A controller contains the business logic and interacts with the database.

✅ 4️⃣ Model na enna?

Model:

Database structure represent pannum

Schema base ah irukum

Mongoose model use pannuvom

Example:

const User = mongoose.model("User", userSchema)

Route → Controller → Model → MongoDB

✅ 5️⃣ Route, Controller & DB Direct Relation Illa Concept

Neenga note pannunga Gowtham:

Route direct ah DB kitta pesa koodathu ❌

Route → Controller → Model → DB

Why?

Clean code

Easy maintain

Large project scale panna easy

Debug easy

Bad example ❌

app.get("/users", async (req,res)=>{ const users = await mongoose.find() })

Good example ✔

Route file:

router.get("/users", getUsers)

Controller file:

export const getUsers = async (req,res)=>{ const users = await User.find() res.json(users) }

👉 Separation of concerns.

English Sentence:
Routes should not directly access the database; controllers handle the business logic.

✅ 6️⃣ Config Folder Purpose

Config folder la:

Database connection logic irukum

Environment setup irukum

Example:

config/db.js

Your code:

import mongoose from "mongoose" const connectDB = async () => { try { await mongoose.connect(process.env.MONGO_URL) console.log("Database Connected Successfully") } catch(error) { console.log("Something Error") } } export default connectDB

Why separate file?

Reusable

Clean structure

Large project manage easy

English Sentence:
The config folder is used to manage configuration like database connections separately.

✅ 7️⃣ Full Flow Diagram (Very Important for Class)

Client (Thunder Client) ↓ Route ↓ Controller ↓ Model ↓ MongoDB ↓ Response back to Client

✅ 8️⃣ MVC Pattern Basic Idea

MVC = Model View Controller

Backend la:

Model → Database structure

Controller → Logic

Route → Entry point

View mostly frontend side.
