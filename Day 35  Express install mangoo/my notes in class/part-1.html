✅ 5️⃣ செயல்கள்

⚡ Important SQL Queries

SQL la 4 basic operations iruku:

INSERT SELECT UPDATE DELETE

🔹 INSERT

INSERT INTO students (name, age) VALUES ('Arun', 20);

👉 Data add pannum.

🔹 SELECT

SELECT * FROM students;

👉 Data edukkum.

🔹 UPDATE

UPDATE students SET age = 22 WHERE id = 1;

👉 Existing data change pannum.

🔹 DELETE

DELETE FROM students WHERE id = 1;

👉 Data remove pannum.

SQL provides commands to insert, select, update, and delete data.
✅ 6️⃣ சிக்கல்கள்

⚠️ SQL Drawbacks (Large Data)

Ipdi yosichu paaru Gowtham 👇

  • Lakhs of records iruntha?
  • Tables romba perusa aana?
  • Fields add panna?
  • Structure change panna?

Problem varum:

  • ❌ Table create pannita apram change panna kashtam
  • ❌ New column add panna ALTER query venum
  • ❌ Relationships maintain panna complex
  • ❌ Large scale apps la slow aagum
SQL becomes complex when handling very large and frequently changing data.
✅ 7️⃣ நெகிழ்வு

🍃 Why NoSQL Introduced?

SQL la rigid structure irukum (strict table design).

So companies enna pannanga?

👉 “Data flexible ah object madhiri store pannalam” nu decide pannanga. Athunala than NoSQL vandhathu.

NoSQL:

  • Table illa
  • Rows illa
  • Columns illa
  • Object format la store pannum
  • Large scale apps ku useful.
NoSQL was introduced to handle flexible and large-scale data efficiently.
