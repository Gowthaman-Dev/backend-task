✅ 1️⃣ Client (Thunder Client / Frontend)

Neenga API test panna use pannura tool:

👉 Thunder Client (VS Code extension)
👉 Illati frontend app

Example:

POST http://localhost:5000/api/users

Client request send pannum.

English Sentence:
The client sends an HTTP request to the server.

✅ 2️⃣ Route – Entry Point

Example:

router.post("/users", createUser)

Route job:

✔ URL match pannum
✔ Correct controller ku anupum

Route DB kitta direct ah pesa koodathu ❌

✅ 3️⃣ Controller – Brain of Backend

export const createUser = async (req, res) => {
   const { name, age } = req.body

   const user = await User.create({ name, age })

   res.status(201).json(user)
}

Controller job:

✔ Request data edukkum
✔ Business logic handle pannum
✔ Model call pannum
✔ Response send pannum

✅ 4️⃣ Model – Database Interaction Layer

Model create pannumbothu:

const User = mongoose.model("User", userSchema)

Controller la:

User.create()
User.find()
User.findById()
User.findByIdAndUpdate()
User.findByIdAndDelete()

Model job:

✔ MongoDB kitta pesum
✔ Data save pannum
✔ Data fetch pannum

✅ 5️⃣ MongoDB – Data Storage

MongoDB:

✔ Document store pannum
✔ BSON format la internally save pannum
✔ _id auto create pannum

Example stored document:

{
  "_id": "656d8fd9f12a8e4c12345678",
  "name": "Arun",
  "age": 22
}

✅ 6️⃣ Response Back to Client

res.json(user)

Server → JSON format la response anupum.

{
  "_id": "656d8fd9f12a8e4c12345678",
  "name": "Arun",
  "age": 22
}

🔥 Full Real Example Flow (Create User API)

Step 1 – Client Request

POST /api/users
Body:
{
  "name": "Arun",
  "age": 22
}

Step 2 – Route

router.post("/users", createUser)

Step 3 – Controller

export const createUser = async (req, res) => {
   const user = await User.create(req.body)
   res.status(201).json(user)
}

Step 4 – Model

User.create()

Step 5 – MongoDB Save

MongoDB la document store aagum.

Step 6 – Response

JSON client ku thirumbi pogum.

✅ 7️⃣ Important Concept – Separation of Responsibility

Layer Responsibility
Route Request receive
Controller Logic handle
Model DB interact
MongoDB Data store
✔ Clean code
✔ Easy debugging
✔ Large project scalable
✔ Team work easy
