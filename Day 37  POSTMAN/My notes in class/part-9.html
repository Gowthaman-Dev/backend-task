🔹 1️⃣ What is Controller?

  • Route request receive பண்ணும்
  • Controller DB work பண்ணும்

🔹 2️⃣ req.body

req.body = Client send பண்ணும் data.

Expressல: req.body

இதுல மேல இருக்குற JSON கிடைக்கும்.

🔹 3️⃣ movieDataModules.create()

Assume model:

import movieDataModules from "../models/movieModel.js"

Create method:

movieDataModules.create(req.body)

👉 Internally:

  • Schema validation
  • Default values apply
  • Timestamps add
  • MongoDB insert

🔹 4️⃣ Why async/await?

Database operation time எடுக்குது.
So it returns Promise.

async (req, res)

await use பண்ணினா:

  • Code readable
  • Proper error handling
  • Clean structure

🔹 5️⃣ try/catch Why?

Database error வந்தா app crash ஆக கூடாது.

try { // DB logic } catch(error) { // Error handle }

🔹 6️⃣ Full Create Controller Example

export const createMovie = async (req, res) => { try { const movie = await movieDataModules.create(req.body) res.status(201).json({ success: true, message: "Movie created successfully", data: movie }) } catch (error) { res.status(404).json({ success: false, message: "Movie not created", error: error.message }) } }

🔹 7️⃣ Status Codes

✅ 201

  • Created Successfully
    Used when new data insert success.

❌ 404

  • Resource Not Found
    (Usually GET use பண்ணுவோம், but simple exampleக்காக இங்கே use பண்ணிருக்கோம்.)

Better practice:

  • 400 → Bad Request
  • 500 → Server Error

🔹 8️⃣ Response JSON Structure (Professional Format)

Always consistent format use பண்ணணும்.

{ "success": true, "message": "Movie created successfully", "data": { "_id": "123456", "name": "Leo", "rating": 9 } }

Why this structure?

  • success → frontend easy handle
  • message → user info
  • data → actual result

🔥 Complete Flow – CREATE API

Client (POST /api/movie/create) ↓ Route ↓ Controller ↓ req.body ↓ Model.create() ↓ Schema Validation ↓ MongoDB Insert ↓ Response JSON (201)
