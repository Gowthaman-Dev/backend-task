✅ Part 12 – Full Architecture Recap

🧠 Big Picture First Client → MongoDB

Client ↓ Route ↓ Controller ↓ Model ↓ Schema ↓ MongoDB

இது தான் Full Backend Architecture 💪

🎯 Goal: Client → Route → Controller → Model → Schema → MongoDB
Full data flow crystal clear ஆகணும் 🔥

இப்போ ஒவ்வொரு step-ஐ detailஆ புரிஞ்சிக்கலாம்.

🔹 1️⃣ Client

👉 Frontend (React / Postman / Thunder Client)

Example request:

POST /api/movie/create

Client என்ன பண்ணுது?

  • URL call பண்ணுது
  • JSON data send பண்ணுது

Example:

{ "movieName": "Forever You", "rating": 9 }

🔹 2️⃣ Route

router.post("/create", createMovie)

Route என்ன பண்ணுது?

  • URL match பண்ணுது
  • Correct controller function call பண்ணுது

👉 Route = Traffic Police 🚦

🔹 3️⃣ Controller

export const createMovie = async (req, res) => { const movie = await movieDataModules.create(req.body) }

Controller என்ன பண்ணுது?

  • req.body receive பண்ணுது
  • Business logic execute பண்ணுது
  • Model call பண்ணுது
  • Response send பண்ணுது

👉 Controller = Brain 🧠

🔹 4️⃣ Model

movieDataModules.create(req.body)

Model என்ன பண்ணுது?

  • MongoDB query create பண்ணுது
  • Schema rules apply பண்ணுது
  • DB-க்கு data send பண்ணுது

👉 Model = Bridge 🌉

🔹 5️⃣ Schema

const movieSchema = new mongoose.Schema({ movieName: String, rating: Number })

Schema என்ன பண்ணுது?

  • Structure define பண்ணுது
  • Validation check பண்ணுது
  • Default values apply பண்ணுது
  • Timestamps add பண்ணுது

👉 Schema = Blueprint 📘

🔹 6️⃣ MongoDB

Final destination 🏁

  • Data store ஆகும்
  • Query process ஆகும்
  • Result return ஆகும்

👉 MongoDB = Storage 🗄️

🔥 Full CREATE Flow Diagram

Client (POST Request) ↓ Express Server ↓ Route match (/api/movie/create) ↓ Controller function run ↓ req.body read ↓ Model.create() ↓ Schema validation ↓ MongoDB insert ↓ Success response (201) ↓ Client receive JSON

🔥 Full GET BY ID Flow

Client (GET /api/movie/12345) ↓ Route (/:userid) ↓ req.params.userid = 12345 ↓ Controller ↓ Model.findById(12345) ↓ MongoDB search ↓ Single document return ↓ Response (200)
