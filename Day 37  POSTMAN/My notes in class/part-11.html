✅ Part 11 – Get By ID (Dynamic Routing)

🔎 ஒரு specific document _id மூலம்

இது CRUDல மிகவும் important 🔥
ஒரு specific document-ஐ அதன் _id மூலம் retrieve பண்ணுவது.

🎯 Goal: Dynamic routing + req.params clear ஆகணும்

🔹 1️⃣ What is Dynamic Routing?

Static route:

/api/movie/get

Dynamic route:

/api/movie/:userid

👉 :userid = Dynamic parameter

இது URLல value change ஆகும்.

Example:

/api/movie/65f2a123abcd1234
/api/movie/88ggs777yyhh9999

Route same தான்
ID மட்டும் change ஆகும் 🔥

🔹 2️⃣ req.params

Dynamic route value எங்கே கிடைக்கும்?

👉 req.params

Example:

req.params.userid

Route:

router.get("/:userid", getMovieById)

Controllerல:

const id = req.params.userid

🔹 3️⃣ findById()

MongoDBல specific document find பண்ண use பண்ணுற method.

movieDataModules.findById(id)

👉 Internally இது _id fieldல search பண்ணும்.

Same as:

movieDataModules.find({ _id: id })

But findById more clean & direct.

🔥 4️⃣ Full Route Example

📁 movieRoute.js

router.get("/:userid", getMovieById)

🔥 5️⃣ Controller Example

export const getMovieById = async (req, res) => { try { const id = req.params.userid const movie = await movieDataModules.findById(id) if (!movie) { return res.status(404).json({ success: false, message: "Movie not found" }) } res.status(200).json({ success: true, data: movie }) } catch (error) { res.status(500).json({ success: false, message: "Server error" }) } }

🔹 6️⃣ Returning Single Field

Sometimes full document வேண்டாம்.

Only one field return பண்ணலாம்.

Example:

const movie = await movieDataModules.findById(id) res.json({ MovieType: movie.MovieType })

OR better way:

const movie = await movieDataModules.findById(id).select("MovieType")

👉 .select() use பண்ணினா specific field மட்டும் வரும்.

Professional method 💯

🔥 7️⃣ Full Flow – Get By ID

Client → GET /api/movie/12345 ↓ Route matches "/:userid" ↓ req.params.userid = 12345 ↓ Model.findById(12345) ↓ MongoDB search _id ↓ Single document return ↓ JSON Response (200)

🔹 8️⃣ Status Codes

  • 200 → Success
  • 404 → ID not found
  • 500 → Server error

🔥 Why Dynamic Routing Important?

Without dynamic routing:

❌ Separate route for every ID create பண்ணணும் 😵

With dynamic routing:

✅ Single route handle all IDs
Scalable + Clean Architecture.

🧠 One Line English Definition:

Dynamic routing allows us to capture variable values from the URL using req.params and retrieve specific documents using findById().
No word change · No missing text · Only HTML CSS no js
