✅ Part 10 – Get All Data + Filter

🔍 READ operation find() + filter

இது CRUDல READ operation 🔥
Data retrieve பண்ணுற முக்கியமான பகுதி.

🎯 Goal: Database level filtering importance clear ஆகணும்.

🔹 1️⃣ Get All Data – find()

movieDataModules.find()

👉 இது collectionல இருக்கும் எல்லா documents-ஐ return பண்ணும்.

Example Controller:

export const getAllMovies = async (req, res) => { try { const movies = await movieDataModules.find() res.status(200).json({ success: true, count: movies.length, data: movies }) } catch (error) { res.status(500).json({ success: false, message: "Failed to fetch movies" }) } }

🔹 2️⃣ JS filter() Method

Assume:

const movies = await movieDataModules.find() const filtered = movies.filter( movie => movie.movieName === "Forever You" )

👉 இது JavaScript filtering.

❌ Why JS filter() is Bad Practice?

Imagine:

  • Databaseல 10,000 records இருக்கு
  • நம்ம find() use பண்ணி எல்லாம் fetch பண்ணுறோம்
  • பிறகு JSல filter பண்ணுறோம்

Problems:

  • 🚨 Performance issue
  • 🚨 Memory waste
  • 🚨 Network bandwidth waste
  • 🚨 Slow response time

Because:

👉 First ALL data appக்கு வரும்
👉 Then filtering நடக்கும்

This is wrong architecture ❌

🔥 3️⃣ Better Way – Database Level Filtering

movieDataModules.find({ movieName: "Forever You" })

👉 MongoDBலேயே filter நடக்கும்.

Example Controller:

export const getFilteredMovies = async (req, res) => { try { const movies = await movieDataModules.find({ movieName: "Forever You" }) res.status(200).json({ success: true, count: movies.length, data: movies }) } catch (error) { res.status(500).json({ success: false, message: "Error fetching filtered movies" }) } }

🔹 4️⃣ Why Database Filtering is Powerful?

🔥 Internally என்ன நடக்குது?

MongoDB Engine ↓ Search only matching records ↓ Send only filtered data

Benefits:

  • ⚡ Faster response
  • 💾 Less memory usage
  • 🌐 Less network traffic
  • 📈 Scalable system

🔥 Real Example Comparison

❌ Wrong Flow
DB → 10,000 records → App → filter() → 1 record
✅ Correct Flow
DB → filter query → 1 record → App

Difference huge 🔥

🔹 5️⃣ Dynamic Filtering (Professional Way)

Instead of hardcoding:

const { movieName } = req.query const movies = await movieDataModules.find({ movieName })

URL:

/api/movie?movieName=Forever You

👉 This is scalable API design.

🔹 6️⃣ Status Codes

  • 200 → Success
  • 404 → Not found (if no records)
  • 500 → Server error

Example:

if (movies.length === 0) { return res.status(404).json({ success: false, message: "No movies found" }) }

🎯 Final Architecture View

Client Request ↓ Controller ↓ Model.find({ condition }) ↓ MongoDB filtering ↓ Filtered data return

🧠 One Line English Definition:

Database-level filtering uses query conditions in find() so that MongoDB returns only matching documents instead of filtering data in JavaScript after retrieval.
