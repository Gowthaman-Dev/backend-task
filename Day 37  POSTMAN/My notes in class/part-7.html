✅ Part 7 – Model Creation

mongoose.model("movieData", movieSchema) இது ரொம்ப முக்கியமான step 🔥 Schema புரிஞ்ச பிறகு next powerful concept = Model

🔹 1️⃣ What is Model?

👉 Model = Controller for Collection

Simple definition:

Model என்பது MongoDB collection-ஐ handle பண்ணும் object.

Schema structure define பண்ணும். Model அந்த structure-ஐ use பண்ணி DB-ல CRUD operations பண்ணும்.

🔹 2️⃣ Why Model After Schema?

Schema → Model → Collection

Reason:

  • Schema → Structure மட்டும் define பண்ணும்
  • Model → DB operations execute பண்ணும்

Schema alone use பண்ண முடியாது.

❌ Wrong:

movieSchema.find() // ❌ error

✅ Correct:

Movie.find() // ✅ working

ஏன்?

Because Model only has DB methods

🔹 3️⃣ mongoose.model() என்ன பண்ணுது?

const Movie = mongoose.model("movieData", movieSchema)

Internally என்ன நடக்குது?

  • mongoose schema-ஐ எடுத்துக்குது
  • "movieData" என்ற collection name உருவாக்குது
  • MongoDB-ல் collection create ஆகுது
  • CRUD methods attach ஆகுது

🔹 4️⃣ Model Acts as Bridge 🔥

Architecture clearா பாரு:

Application (Node/Express) ↓ Model ↓ MongoDB

Explanation:

  • Application → Request அனுப்பும்
  • Model → Query convert பண்ணும்
  • MongoDB → Data store / retrieve பண்ணும்

👉 Model தான் Translator.

🔹 5️⃣ Example Full Code

import mongoose from "mongoose" const movieSchema = new mongoose.Schema({ name: String, rating: Number }) const Movie = mongoose.model("movieData", movieSchema) export default Movie

🔹 6️⃣ Collection Name Logic

mongoose.model("movieData", movieSchema)

MongoDBல collection name automatically:

moviedatas

👉 Mongoose:

  • Lowercase ஆக்கும்
  • Plural ஆக்கும்

🔹 7️⃣ Model Methods (Power Area 💪)

Model use பண்ணி:

Movie.create() Movie.find() Movie.findById() Movie.findByIdAndUpdate() Movie.findByIdAndDelete()

All DB operations model மூலம் தான்.

🎯 Final Architecture View

Client ↓ Route ↓ Controller ↓ Model ↓ MongoDB

🧠 One Line English Definition:

A Model is a wrapper around a MongoDB collection that allows the application to interact with the database using the defined schema.
