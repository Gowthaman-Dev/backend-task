✅ Part 8 – Route Mounting movie.use("/api/movie", movieroute)

இது Backend architectureல ரொம்ப முக்கியமான concept 🔥
இது புரிஞ்சா API structure clean ஆகும்.

🔹 1️⃣ What is Route Mounting?

👉 Route Mounting = Base URL க்கு route file attach பண்ணுவது

Simpleஆ சொன்னா:

Main app-ல ஒரு base path குடுத்து, அந்த path க்கு route file connect பண்ணுவது தான் route mounting.

movie.use("/api/movie", movieroute)

இதன் அர்த்தம்:

  • /api/movie = Base URL
  • movieroute = Separate route file

இப்போ movieroute உள்ளே இருக்கும் எல்லா routes கும்
/api/movie automatically prefix ஆகும்.

🔹 2️⃣ Why Separate Route File?

❌ எல்லா routes-ஐயும் server.jsல எழுதினா:

  • Code messy ஆகும்
  • Maintain பண்ண கஷ்டம்
  • Big project handle பண்ண முடியாது

✅ Separate file use பண்ணினா:

  • Clean structure
  • Easy debugging
  • Team work easy
  • Scalable architecture
Structure Example:
server.js
routes/
movieRoute.js
userRoute.js
controllers/
models/

Professional backend இப்படித்தான் இருக்கும் 💪

🔹 3️⃣ Why express.Router()?

import express from "express"

const router = express.Router()

👉 Router = Mini Express App
Main app மாதிரி தான் work பண்ணும்
ஆனா separate moduleல.

Why not use app directly?

❌ app use பண்ணினா எல்லா routes main fileல தான் இருக்கணும்.
✅ Router use பண்ணினா modular structure கிடைக்கும்.

🔹 4️⃣ Route File Example

📁 movieRoute.js

import express from "express"
const router = express.Router()

router.post("/create", (req, res) => {
res.send("Movie Created")
})

router.get("/get", (req, res) => {
res.send("Movie List")
})

export default router

🔹 5️⃣ Mounted in server.js

import movieroute from "./routes/movieRoute.js"

movie.use("/api/movie", movieroute)

🔹 6️⃣ Final URL Structure

Because of mounting:

POST /api/movie/create
GET /api/movie/get

Internally:

Base Path + Route Path
/api/movie + /create
─────────────
/api/movie/create

🔥 Full Flow Architecture

Client Request
↓
Server (app.use)
↓
Mounted Route File
↓
Specific Route
↓
Controller
↓
Model
↓
MongoDB

🔹 Why "/api" Prefix?

Best practice 💯

  • /api → Backend endpoints
  • / → Frontend
  • Version control possible → /api/v1/movie

Professional REST API structure இப்படித்தான் இருக்கும்.
